# Инструкции по выполнению лабораторной работы

## Цель работы
Научиться использовать Node.js для взаимодействия с PostgreSQL базой данных, реализовать основные CRUD операции и понять принципы работы с базами данных в приложениях на Node.js.

## Подготовка к работе

1. Убедитесь, что у вас установлены:
   - Node.js (версия 14 или выше)
   - PostgreSQL (версия 10 или выше)
   - Git (для клонирования репозитория, если применимо)

2. Установите зависимости проекта:
   ```bash
   npm install
   ```

3. Настройте подключение к базе данных в файле `.env`:
   ```
   DB_HOST=localhost
   DB_PORT=5432
   DB_NAME=lab_db
   DB_USER=postgres
   DB_PASSWORD=postgres
   ```

## Запуск проекта

1. Сначала инициализируйте базу данных:
   ```bash
   npm run init-db
   ```

2. Затем запустите основное приложение:
   ```bash
   npm start
   ```

## Структура проекта

- `index.js` - основной файл приложения, запускает примеры использования моделей
- `init-db.js` - скрипт для создания базы данных и таблиц
- `models/` - директория с моделями для работы с данными
- `package.json` - файл с зависимостями и скриптами проекта
- `.env` - файл с настройками подключения к базе данных
- `examples.sql` - файл с примерами SQL-запросов
- `concepts.md` - файл с теоретическими концепциями

## Основные задания для лабораторной работы

1. **Изучение структуры проекта** - ознакомьтесь с файлами и папками проекта
2. **Анализ модели User** - изучите файл `/models/User.js` и понимайте, как реализованы CRUD операции
3. **Добавление новых функций** - реализуйте дополнительные методы в модели User или создайте новые модели
4. **Работа с SQL** - изучите и модифицируйте примеры из файла `examples.sql`
5. **Тестирование** - запустите приложение и проверьте его работу

## Возможные расширения

1. Создание дополнительных моделей (например, для хранения заказов, товаров и т.д.)
2. Реализация связей между таблицами (один ко многим, многие ко многим)
3. Добавление валидации данных
4. Создание простого API с использованием Express.js
5. Реализация транзакций для обеспечения целостности данных
