# Основные концепции баз данных

## Что такое база данных?
База данных — это организованная коллекция структурированных записей или данных, которая обычно хранится в компьютерной системе. База данных обычно контролируется системой управления базами данных (СУБД).

## Что такое PostgreSQL?
PostgreSQL — это свободно распространяемая объектно-реляционная система управления базами данных (ORDBMS), которая расширяет возможности SQL для поддержки большого количества современных приложений.

## Основные SQL команды

### DDL (Data Definition Language) - команды определения данных:
- `CREATE` - создание объектов базы данных
- `ALTER` - изменение структуры существующих объектов
- `DROP` - удаление объектов базы данных
- `TRUNCATE` - удаление всех записей из таблицы

### DML (Data Manipulation Language) - команды манипуляции данными:
- `SELECT` - выборка данных из базы данных
- `INSERT` - добавление новых данных
- `UPDATE` - обновление существующих данных
- `DELETE` - удаление данных

## Основные принципы проектирования баз данных

### Нормализация
Процесс организации данных в базе данных с целью уменьшения избыточности и улучшения целостности данных.

### Первичный ключ (Primary Key)
Уникальный идентификатор для каждой строки в таблице. Не может быть NULL и должен быть уникальным.

### Внешний ключ (Foreign Key)
Поле (или набор полей), которое используется для установления связи между таблицами. Указывает на первичный ключ другой таблицы.

## Node.js и PostgreSQL
Node.js позволяет взаимодействовать с PostgreSQL через различные библиотеки, такие как 'pg', 'pg-promise', 'sequelize' и другие. Библиотека 'pg' является официальным драйвером для PostgreSQL в Node.js.

## CRUD операции
CRUD — это аббревиатура, обозначающая четыре основные операции при работе с данными:
- **C**reate (создание) - INSERT
- **R**ead (чтение) - SELECT
- **U**pdate (обновление) - UPDATE
- **D**elete (удаление) - DELETE
